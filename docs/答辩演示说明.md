# InsightPilot 答辩演示说明

## 一、推荐演示顺序（8-10 分钟）

1. 介绍选题背景：传统数据分析门槛高，大模型可降低自然语言交互门槛。
2. 展示注册登录和首页统计卡片。
3. 上传 `sample_data/student_learning.csv`。
4. 展示健康度、7 个缺失单元格、2 条重复数据和异常值检测结果。
5. 执行“删除重复数据”或“均值填补缺失值”。
6. 在统计工作台执行“按专业统计平均成绩”。
7. 在 AI 助手输入同样的问题，展示 JSON 计划、真实结果、图表和解释。
8. 说明安全设计：不使用 `exec()`，只允许白名单动作。
9. 生成报告并导出 HTML。
10. 展示测试结果和项目目录。

## 二、常见设问

### 为什么不让大模型直接计算？

大模型可能产生幻觉，统计数字必须由 Pandas 或 Scikit-learn 根据真实数据计算，大模型只负责意图理解和文字解释。

### 为什么使用白名单？

避免模型生成危险 Python、SQL 或文件操作指令。后端只执行预先实现且可验证的分析动作。

### 数据为什么不全部存入 MySQL？

数据库保存元数据和业务记录，原始文件保存在文件系统，结构简单且适合课程项目规模。

### 没有大模型密钥能否演示？

可以。规则模式无需网络，支持常见中文分析问题；配置兼容接口后可切换真实大模型。

### 项目和数据分析课程有什么关系？

项目使用了教材中的数据载入、缺失值处理、重复值处理、分组聚合、描述统计、标准化、相关分析、回归、聚类和可视化知识。
