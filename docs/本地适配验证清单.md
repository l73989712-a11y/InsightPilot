# InsightPilot 本地适配验证清单

## 一、基础环境

- [ ] Python 版本为 3.10 或 3.11。
- [ ] `python --version` 和 `pip --version` 可以正常执行。
- [ ] 已创建并激活虚拟环境。
- [ ] `pip install -r requirements.txt` 无报错。
- [ ] `python scripts/check_environment.py` 中依赖与目录检查通过。

## 二、MySQL 5.7

- [ ] MySQL 服务已经启动。
- [ ] 能使用 `mysql -uroot -p` 登录。
- [ ] 已执行 `sql/init.sql`。
- [ ] 数据库名称为 `insightpilot`。
- [ ] `.env` 中数据库用户名、密码、主机和端口正确。
- [ ] `flask --app run.py init-db` 执行成功。

## 三、Web 功能

- [ ] `python run.py` 能启动服务。
- [ ] 浏览器能访问 `http://127.0.0.1:5000`。
- [ ] 注册、登录、退出功能正常。
- [ ] CSV 文件上传成功。
- [ ] Excel 文件上传成功。
- [ ] 数据预览和字段质量信息正常显示。
- [ ] 数据清洗后行数或字段值按预期变化。
- [ ] 统计分析结果表格正常显示。
- [ ] 柱状图、折线图、散点图或热力图正常显示。
- [ ] AI 规则模式能够回答常见分析问题。
- [ ] HTML 报告可以生成、打开和下载。

## 四、自动化检查

在项目根目录执行：

```bash
python -m pytest -q
python -m compileall app scripts tests run.py config.py
```

- [ ] 测试全部通过。
- [ ] 未出现 Python 语法错误。

## 五、真实大模型接口（可选）

- [ ] `.env` 中 `LLM_PROVIDER=openai_compatible`。
- [ ] `LLM_API_KEY` 已填写。
- [ ] `LLM_BASE_URL` 以兼容接口的 `/v1` 路径结尾。
- [ ] `LLM_MODEL` 与服务端支持的模型名一致。
- [ ] 网络能够访问模型接口。
- [ ] AI 问答失败时，系统仍可切回 `rule` 模式演示。

## 六、报告和提交材料

- [ ] 已填写姓名、学号、班级、分工等内容。
- [ ] 实训报告 DOCX 可以正常编辑。
- [ ] 实训报告 PDF 为 A4 版式且页数不少于 20 页。
- [ ] 项目源代码、SQL、样例数据、文档均在最终压缩包中。
- [ ] `.env` 未包含在公开提交包中。
- [ ] 答辩前使用本机重新截取关键页面。

## 七、出现问题时优先检查

1. 数据库连接错误：核对 MySQL 服务、密码、端口、数据库名和字符集。
2. 页面样式缺失：确认 `app/static/vendor` 文件完整。
3. 图表不显示：查看浏览器开发者工具 Console，确认 Plotly.js 已加载。
4. Excel 读取失败：重新安装 `openpyxl` 和 `xlrd`。
5. 中文 CSV 乱码：将文件保存为 UTF-8、UTF-8 with BOM、GBK 或 GB18030。
6. AI 接口失败：先切换到规则模式，再检查密钥、地址、模型名和网络。
