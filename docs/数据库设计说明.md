# InsightPilot 数据库设计说明

## 1. 设计原则

数据库保存用户、数据集元数据、清洗记录、分析任务、AI 会话和报告。原始 CSV/Excel 文件保存在文件系统中，避免将大文件直接写入数据库。

## 2. 核心数据表

| 表名 | 作用 |
|---|---|
| users | 用户账号和密码摘要 |
| datasets | 数据集名称、路径、规模、字段信息和质量报告 |
| cleaning_records | 清洗操作、参数和处理前后行数 |
| analysis_tasks | 分析类型、参数、真实结果和图表配置 |
| ai_conversations | AI 会话标题及所属数据集 |
| ai_messages | 用户问题、助手回答和分析计划 |
| reports | 自动生成的 HTML 报告 |

## 3. 关系说明

一个用户可以拥有多个数据集；一个数据集可以产生多条清洗记录、分析任务、AI 会话和报告；一个 AI 会话包含多条消息。外键设置级联删除，删除数据集时同步删除其业务记录。

## 4. 兼容性

字段中的 JSON 内容使用 LONGTEXT 保存，不依赖 MySQL 原生 JSON 类型，便于适配 MySQL 5.7 及课程要求的版本范围。
